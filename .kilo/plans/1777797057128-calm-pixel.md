# Plan: Match Admin Page Width to Header Width and Add New Module Creation Page

## Problem
1. The Header component has a specific width constraint (max-w-7xl with responsive padding) while the Admin page uses a generic container class that may not match the Header's width exactly.
2. Creating new modules currently happens through a wizard in the AdminModuleEditor component, but users want a dedicated page for creating new modules.
3. There is a build error in src/components/admin-module-editor.tsx: "Return statement is not allowed here" due to incorrect placement of the getStatusColor function causing a syntax error where the main return statement appears to be outside the function body.

## Solution
1. Modify the Admin page container to use the same width and padding classes as the Header component.
2. Create a new page route at `/admin/modules/new` for creating new modules.
3. Add navigation from the AdminModuleEditor to the new module creation page.
4. Fix the build error in admin-module-editor.tsx by correcting the getStatusColor function placement and syntax.

## Changes Needed
1. Update the container div in `src/app/admin/page.tsx` to match the Header's container classes:
   - Change from: `<div className="container mx-auto px-4 py-8">`
   - Change to: `<div className="mx-auto max-w-7xl px-4 sm:px-6 lg:px-8 py-8">`
2. Create new file `src/app/admin/modules/new/page.tsx` for the new module creation page
3. Update `src/app/admin/page.tsx` to remove the New Module button that opens the wizard (replaced by navigation)
4. Update `src/app/admin/components/admin-module-editor.tsx` to navigate to the new page instead of opening the wizard
5. Fix the build error in admin-module-editor.tsx by:
   - Moving the misplaced getStatusColor function to the correct location
   - Ensuring proper function syntax with return type
   - Ensuring all helper functions are properly declared within the component function

## Implementation Details
- The Header container uses: `mx-auto max-w-7xl px-4 sm:px-6 lg:px-8`
- We'll apply the same classes to the Admin page main container
- Keep the `py-8` padding for vertical spacing (already present)
- The new module creation page will reuse the ModuleWizard component but in a full-page layout
- Navigation will use Next.js router to go to `/admin/modules/new`
- Fix the component structure by properly declaring the getStatusColor function after filteredModules and before handleDeleteModule
